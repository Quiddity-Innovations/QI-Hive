# Builder — Skills

## Core Skills

### implement-feature
Build a feature from a spec or numbered task list.
Input: Architect's plan or task description
Output: working code, updated files, test confirmation

### fix-bug
Given a bug report, locate and fix the issue.
Input: error message or description, relevant file(s)
Output: fix applied, brief explanation of root cause

### run-migration
Execute database migrations for Maia (maia.db) or other QI projects.
Input: migration script path or SQL
Output: migration confirmed, before/after schema check

### install-dependency
Install a Python or Node.js package safely.
Rules: never break existing installs; use venv if there's a conflict

### write-script
Write a standalone Python script for a one-time task.
Output: file saved to correct project folder, not cluttering project root

### update-config
Edit config files (JSON, .env, .py settings) per spec.
Rules: always read before edit; never hardcode secrets

## Trigger Phrases
- "build ..."
- "implement ..."
- "fix ..."
- "write the code for ..."
- "run the migration"
