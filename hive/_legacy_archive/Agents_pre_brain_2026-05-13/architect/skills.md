# Architect — Skills

## Core Skills

### design-feature
Produce a step-by-step implementation plan for a new feature.
Input: feature description, affected project, constraints
Output: numbered task list, file-by-file changes, risk flags

### design-integration
Plan how two QI projects connect via API.
Input: project A, project B, integration goal
Output: endpoint spec, data contract, port assignments, CORS rules

### write-adr
Write an Architecture Decision Record (ADR).
Format: Context → Decision → Consequences → Alternatives considered

### review-design
Review a proposed design before build starts.
Output: approved / rejected / needs-revision + reasoning

### port-check
Before any new service: verify port is in allocated block for the project.
Reference: C:\QI\ECOSYSTEM\qi_registry.json

## Trigger Phrases
- "design a ..."
- "plan how to ..."
- "should we use ..."
- "what's the architecture for ..."
- "review this design"
