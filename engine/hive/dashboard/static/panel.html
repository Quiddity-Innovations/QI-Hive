1. Start the panel on the home machine

  1. On the PowerSpec, double-click Start QI Panel on the Desktop.
  2. Wait for the terminal to finish. It will:
    • Start all required Windows services (Dashboard, Tunnel, Maia, Naya, Link Collector)
    • Refresh links.json with the latest tunnel URLs
    • Save the panel URL to Desktop\QI_PANEL_URL.txt + copy to clipboard
    • Open the panel in your default browser
  3. You should see the panel load. Confirm the top strip shows live tunnels count, e.g. 4 live tunnels · 10 LAN-only.

2. Verify each LIVE tunnel works

  1. For every card with a green LIVE badge, click Open (public).
  2. The app should open in a new tab. If it does — close the tab, move to the next.
  3. If a LIVE link fails to load, the tunnel rotated mid-check. Come back to the panel and click ⟳ Refresh links.
If a card you expect to be LIVE shows LAN ONLY, its underlying tunnel service is down. Open a terminal and run nssm status QI_MaiaTunnel (or the relevant service). Restart it with gsudo nssm restart QI_<Name>.

3. Send the panel URL to the other machine

  1. The URL is already on your clipboard. Pick one delivery method:
    • Text it to yourself (paste into Signal / SMS)
    • Email it to yourself
    • Open Desktop\QI_PANEL_URL.txt and copy from there
  2. On the other machine, paste the URL into any browser → you'll see the same panel.

4. Use the panel remotely

  1. Click any Open (public) button → the app opens in a new tab on that machine.
  2. Cards marked LAN ONLY are only reachable from the home network — their Localhost buttons won't work on a remote machine. That's expected.
  3. Every 30 seconds, the panel auto-refreshes the link list. You can also click ⟳ Refresh links manually.
  4. Use Copy all URLs to grab every live link at once (e.g. to share or save).

5. If a tunnel rotates mid-session

Cloudflare quick tunnels can rotate (rare, but possible). When that happens:

  1. Click ⟳ Refresh links. The collector reads the new URL from the tunnel log; the panel re-renders with the new URL within 30 seconds.
  2. If the panel's own URL (the dashboard tunnel) rotated, the page itself stops loading. Get the new URL from the home machine: re-run Start QI Panel and use the new URL it prints.

6. Troubleshooting

7. Closing down

Nothing to close — services run 24/7. Just close the browser tabs when you're done. The PowerSpec keeps everything alive.

All services start automatically with Windows. You only need to run Start QI Panel when you specifically want to know the current public URL or need to nudge a stopped service back to life.
