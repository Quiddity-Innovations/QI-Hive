## QI Project Standard

Every QI project must have a `docs/` folder at its root containing all
human-readable documentation. Subfolder grouping is optional but recommended:

```
docs/
├── README.md              ← this file (auto-generated)
├── user_guide.docx        ← end-user documentation
├── technical_reference.*  ← architecture + API
├── operator_guide.*       ← deployment + operations
├── implementation_log.*   ← chronological dev log
├── meeting_minutes.*
├── version_history.*
├── archive/               ← old / superseded docs
├── business/              ← pitch decks, business plans
└── external/              ← third-party resources
```
